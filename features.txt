Warranty Management System Implementation Roadmap

Project Goal
Build the current product and warranty CRUD project into a complete business-ready platform with customer workflows, admin control, stronger backend rules, and a polished frontend experience.

Current Base
- Authentication
- Product catalog management
- Warranty registration
- Warranty lookup by serial number

Roadmap Strategy
- Phase 1 focuses on stability and core usability.
- Phase 2 adds real business logic.
- Phase 3 adds service integrations and stronger product value.
- Phase 4 adds premium demo features for a final-year or portfolio project.

Phase 1: Foundation and Cleanup
Goal: Make the existing system reliable, secure, and clean before adding larger features.

Backend tasks in warranty-management
- Move database password out of application.properties into environment variables.
- Add DTOs for auth, product, and warranty responses instead of returning entities directly.
- Add global exception handling using @ControllerAdvice.
- Add validation annotations like @NotBlank, @NotNull, @Email, @Min.
- Add proper error messages for invalid serial number, missing product, duplicate email, and duplicate model number.
- Normalize input values like model number, email, and serial number.
- Add unit tests for:
  - warranty expiry calculation
  - login failure
  - duplicate product prevention
  - duplicate serial number prevention

Frontend tasks in warranty-ui
- Replace alert() with toast notifications.
- Add loading states for login, registration, product save, and warranty search.
- Add empty states for no products and no search result.
- Improve result cards with clear active and expired styles.
- Add form validation messages before backend submission.

Deliverables
- Safer backend
- Cleaner error handling
- Better user feedback
- Stable codebase for future features

Phase 2: Core Business Logic
Goal: Add the features that make this a true warranty platform instead of a CRUD app.

Feature 1: My Warranties Dashboard
Backend
- Add endpoint to fetch all warranties for logged-in customer.
- Add summary counts:
  - active
  - expired
  - claimed
  - expiring soon

Frontend
- Create My Warranties page.
- Add filters by status.
- Add card or table layout for registered warranties.
- Add search by serial number or product name.

Feature 2: Warranty Claim Workflow
Backend
- Create Claim entity.
- Add claim states:
  - REQUESTED
  - UNDER_REVIEW
  - APPROVED
  - REJECTED
  - CLOSED
- Add claim service and claim controller.
- Add rules:
  - expired warranty cannot claim
  - already claimed warranty cannot open duplicate claim
  - only owner or admin can view claim

Frontend
- Add claim request form.
- Add claim status badge and timeline.
- Add claim list page for admin and customer.

Feature 3: Warranty Registration Rules
Backend
- Purchase date cannot be in future.
- Product must exist before warranty creation.
- User must exist before warranty creation.
- Serial number must be unique.
- Purchase date older than allowed threshold should need admin approval.

Frontend
- Validate purchase date.
- Format serial input cleanly.
- Show rule-based error messages.

Deliverables
- Customer dashboard
- Claim module
- Stronger registration rules

Frontend

Deliverables

Architecture Improvements To Keep In Mind
- Separate entity, DTO, service, and controller responsibilities clearly.
- Avoid exposing passwords or internal entity structures in API responses.
- Use service-layer business rules instead of controller-side logic.
- Add reusable utility methods for status calculation and expiry checks.
- Later, consider pagination for large product and warranty lists.
